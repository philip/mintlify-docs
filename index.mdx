## Quickstart prompts

Copy a setup prompt to your AI assistant, or browse our [framework](/get-started/frameworks), [language](/get-started/languages), and [ORM](/get-started/orms) guides.

<Prompt description="Connect **Next.js** to Neon Postgres" actions={["copy", "cursor"]}>
You are an expert software agent specializing in TypeScript and the Next.js framework. Configure the current Next.js project to connect to a Neon Postgres database.

Steps:
1. Prompt the user to select a PostgreSQL driver: @neondatabase/serverless (recommended for serverless/edge), postgres (postgres.js for long-running servers), or pg (node-postgres).
2. Install the selected driver.
3. Configure DATABASE_URL in .env.local — prompt the user for their Neon connection string (found in Neon Console → Connect).
4. Create a centralized database module at app/lib/db.ts.
5. Create examples: Server Component (app/page.tsx), Server Action (app/action/page.tsx), API Route (app/api/version/route.ts).
6. Start the dev server and verify all three patterns work.

Do not hardcode credentials. Do not output the user's connection string. Apply changes directly to the codebase.
</Prompt>

<Prompt description="Connect **Django** to Neon Postgres" actions={["copy", "cursor"]}>
You are an expert software agent responsible for configuring the current Django project to connect to a Neon Postgres database.

Steps:
1. Install required packages: psycopg2-binary (or psycopg[binary]), dj-database-url, python-dotenv.
2. Configure DATABASE_URL in .env — prompt the user for their Neon connection string (found in Neon Console → Connect).
3. Update settings.py to load DATABASE_URL using dj-database-url and python-dotenv.
4. Create a sample Django app with a model, view, and URL pattern to verify the connection.
5. Run migrations and start the dev server.

Do not hardcode credentials. Do not output the user's connection string. Apply changes directly to the codebase.
</Prompt>

<Prompt description="Integrate **Drizzle ORM** with Neon Postgres" actions={["copy", "cursor"]}>
You are an expert software agent responsible for configuring the current TypeScript/Node.js project to connect to a Neon Postgres database using Drizzle ORM.

Steps:
1. Install packages: drizzle-orm, @neondatabase/serverless, drizzle-kit.
2. Configure DATABASE_URL in .env — prompt the user for their Neon connection string (found in Neon Console → Connect).
3. Create drizzle.config.ts with the Neon driver configuration.
4. Create a database schema file with a sample table.
5. Create db.ts module using neon() from @neondatabase/serverless with drizzle-orm/neon-http.
6. Generate and run migrations with drizzle-kit.
7. Create a demo script showing CRUD operations.

Do not hardcode credentials. Do not output the user's connection string. Apply changes directly to the codebase.
</Prompt>
